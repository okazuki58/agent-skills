# issue-create

雑な要件や思考メモから GitHub Issue を構造化して起票するスキル。

## 使い方

Claude Code で `/issue-create` と入力し、要件を伝えるだけ。

```
> /issue-create
見積編集画面に消費税率カラムを追加したい。操作列のUIも整理したい。
```

不明点があれば質問してくれるので、答えるだけで構造化された Issue が作成されます。

## 出力される Issue の構造

```markdown
## 背景・目的
## 要件
### やること
### やらないこと
## 完了条件（DoD）
## 未確定事項・要確認事項
```

## 特徴

- **推測しない** — メモに書かれていない仕様は勝手に決めず、必ず質問する
- **コードは書かない** — Issue の起票に専念し、技術選定や実装方針には踏み込まない
- **完了条件を明確化** — チェックリスト形式の DoD を自動生成

## 前提条件

- GitHub CLI (`gh`) がインストール・認証済み
- git リポジトリ内で実行すること
