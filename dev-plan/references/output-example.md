# 出力例

以下は `docs/prompts/large-project.md` の実例に基づく出力フォーマット。

## フォーマット

````markdown
# {プロジェクト名} — Claude Code 実行プロンプト

## 実行順序

```
Wave 1（直列）: #2 → #3
Wave 2（並列）: #4, #5, #6
Wave 3（並列）: #7, #8
```

各Waveの前のWaveのPRが{ベースブランチ}にマージ済みであること。

---

## Wave 1（直列）

### #{番号} {Issueタイトル}

```
/gtr-workflow で {ベースブランチ} ブランチから {ブランチ名} ブランチを作成して作業して。

まず gh issue view {番号} でIssue内容を読んで。
補足として docs/SPEC-xxx.md を参照して。

Issue #{番号} の完了条件（DoD）を全て満たすように実装して。
実装が完了したら /e2e-test スキルを使ってE2Eテストを書き、テストが全てパスすることを確認して。
確認できたら /pull-request スキルを使って {ベースブランチ} に向けてPRを作成して。コミットメッセージに "closes #{番号}" を含めて。
```

---

## Wave 2（並列 3本）

### #{番号} {Issueタイトル}

```
（同じ形式で各Issueのプロンプトを配置）
```
````

## Wave表記ルール

- **直列**: `#A → #B` — Aの完了後にBを開始
- **並列**: `#A, #B, #C` — 同時に開始可能
- **並列 N本**: Wave内のIssue数を見出しに含める（例: `Wave 2（並列 3本）`）
- 直列Waveでも1本だけなら「直列」表記は不要（例: `Wave 3` のみ）
