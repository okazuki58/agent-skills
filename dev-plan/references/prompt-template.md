# プロンプトテンプレート

各Issueに対して以下のテンプレートでプロンプトを生成する。
`{...}` は実際の値に置き換える。省略可能な行は該当情報がない場合に削除する。

## テンプレート

````
```
/gtr-workflow で {ベースブランチ} ブランチから {ブランチ名} ブランチを作成して作業して。

まず gh issue view {Issue番号} でIssue内容を読んで。
{補足ドキュメント行（省略可能）}
{既存コード参考行（省略可能）}

Issue #{Issue番号} の完了条件（DoD）を全て満たすように実装して。
実装が完了したら /e2e-test スキルを使ってE2Eテストを書き、テストが全てパスすることを確認して。
確認できたら /pull-request スキルを使って {ベースブランチ} に向けてPRを作成して。コミットメッセージに "closes #{Issue番号}" を含めて。
```
````

## 補足ドキュメント行の生成ルール

Issue本文中に `docs/` や `SPEC-` への参照がある場合に生成する。

パターン例:
- `補足として docs/SPEC-rough-estimate.md を参照して。`
- `補足として docs/SPEC-rough-estimate.md の「カテゴリ整理画面」セクションを参照して。`
- `補足として docs/SPEC-rough-estimate.md と docs/SPEC-final-estimate.md を参照して。`

## 既存コード参考行の生成ルール

Issue本文中に既存コードへの言及がある場合に生成する。

パターン例:
- `既存の app/projects/ のコードを参考にして、同様のパターンで作成して。`
- `既存の estimate-table.tsx を読んでパターンを把握してから実装して。`
- `現行の app/projects/[id]/memo/ のコードを読んで、同じパターンで実装して。`

## 特別な指示の追加

Issue本文に実装上の注意点（技術的制約、特定のライブラリ使用指示等）がある場合は、
DoD行の前に追加行として挿入する。

例:
```
矢印の描画は SVG で実装して。左右のアイテムの位置を計算し、パスで接続する形。
```
