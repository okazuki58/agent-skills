# pull-request

コミット済みの変更内容を分析し、簡潔でわかりやすい PR を自動作成するスキル。

## 使い方

```
> /pull-request
```

または Issue 番号を指定して：

```
> PR作成して。Issue #123 に関連
```

## 処理の流れ

1. `git diff develop..HEAD` で変更内容を分析
2. PR タイトル・本文を自動生成
3. ブランチをリモートにプッシュ
4. `gh pr create` で PR を作成

## 生成される PR の構造

```markdown
## 関連する issue
close #123

## 概要
{1〜2文で簡潔に}

## 背景
{問題の原因と修正理由}

## 動作確認
### 自動テスト
### 手動テスト
#### 操作手順
1. ...

#### 期待する結果
- ...

## その他
**変更ファイル:**
- ファイル名 (変更内容)
```

## 特徴

- **簡潔さ重視** — 概要と背景は 2〜3 文で
- **再現可能な手動テスト** — 操作手順を番号付きで詳細に記載
- **誰でも読める** — 専門用語を最小限に

## 前提条件

- コミットが完了していること
- GitHub CLI (`gh`) がインストール・認証済み
- 現在のブランチが develop / main 以外であること
